此區塊說明本應用的核心價值與三大設計支柱。點擊下方卡片以深入了解每項原則背後的邏輯與優勢。

Core Position
輕量化、跨裝置、高度自定義的遊戲輔助儀表板

為了解決遊戲中多種統計資料（如食譜、市場、核芯）分散在不同試算表或網頁的問題，我們將零散的資訊連結整合進一個美觀、具備管理權限的單一入口。

🧩

A. 模組化設計

將每一個功能視為獨立卡片。

方便未來擴展新功能（如：莊園佈置、營地管理），而不需改動整體框架，確保系統高可擴充性。
🔐

B. 雙層權限機制

訪客與管理員雙模式切換。

訪客僅能瀏覽，介面乾淨防誤觸；管理員透過隱藏入口登入，解鎖雲端參數的動態調整能力。
☁️

C. 雲端同步與即時性

跨裝置無縫切換，資料持久化。

不使用 LocalStorage，全面採用 Firestore 儲存自訂連結，確保 PC 修改後手機端立即生效。

⚙️ 技術架構建議

為了達成「輕快」與「雲端同步」的要求，本系統採用無伺服器架構。將滑鼠懸停於各層級，查看其負責之工作與選型理由。

1. 前端框架層 (Frontend)

HTML5 + Tailwind + JS

負責所有視覺渲染與邏輯互動。

選型理由：遊戲輔助工具首重「快」，Tailwind 能確保 CSS 體積最小且快速開發。原生 JS 即可滿足動態 DOM 生成需求。
API 雙向綁定

2. 身份驗證層 (Authentication)

Firebase Anonymous Auth

管理權限與隔離資料空間。

核心目的：確保每個使用者即使不註冊帳號，也能在系統內擁有獨立的資料空間。
安全規則防護

3. 資料庫與儲存層 (Database)

Firebase Firestore

儲存所有自訂連結配置陣列。

資料結構：
Collection: artifacts → Document: appId → Field: linkData (Array)
